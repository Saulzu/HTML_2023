Comandos Git

¿Qué es Git?

Git es un sistema de control de versiones de software diseñado para manejar proyectos grandes y con muchos colaboradores. Permite registrar los cambios realizados en los archivos de un proyecto a lo largo del tiempo, de manera que sea posible revertir fácilmente a versiones previas en caso de necesidad.

      init

      El comando init se utiliza para inicializar un repositorio Git en un directorio específico. Este comando crea una estructura de archivos en el directorio que se usará para registrar los cambios realizados en los archivos del proyecto.

      status

      El comando status muestra el estado actual de los archivos en el repositorio. Indica cuáles son los archivos nuevos o modificados y cuáles están en seguimiento o ignorados por Git.

      branch

      El comando branch permite crear, listar y eliminar ramas (branches) en un repositorio Git. Las ramas son copias separadas del código principal que permiten desarrollar funcionalidades nuevas sin afectar el código principal. Al final, se pueden unir las ramas al código principal a través de un merge.

      checkout

      El comando checkout permite cambiar entre ramas o versiones específicas del código en un repositorio.

      pull

      El comando pull es una forma de descargar y fusionar cambios desde un repositorio remoto en el repositorio local. Este comando ejecuta "git fetch" y "git merge" en un solo paso para mantener el repositorio local actualizado con el remoto.

      push

      El comando push se utliza para enviar cambios desde el repositorio local a un repositorio remoto. Este comando envía las confirmaciones (commits) del repositorio local al repositorio remoto, actualizando así el contenido en el repositorio remoto. Es una forma de compartir cambios con otros desarrolladores y mantener una copia de seguridad en un repositorio remoto.

      clone

      El comando clone se utiliza ara crear una copia local de un repositorio remoto. Este comando descarga todo el contenido del repositorio remoto en una nueva carpeta en el equipo local, incluyendo todas las versiones y ramas del repositorio. Con "git clone", se puede trabajar localmente en un proyecto y luego enviar los cambios de vuelta al repositorio remoto mediante "git push". Es una forma de obtener una copia de un proyecto existente y colaborar con otros desarrolladores.

      commit

      El comando commit se utiliza para registrar cambios en el repositorio local. Este comando guarda una instantánea de los archivos actuales en el repositorio, incluyendo los cambios realizados en ese momento. Al hacer un "commit", se puede describir los cambios realizados en un mensaje de confirmación, lo que permite un seguimiento de las versiones del proyecto y una revisión posterior de los cambios. Los "commits" son la base de la gestión de versiones en Git y son una parte esencial del flujo de trabajo en Git.
